# 贵金属价格监控策略文档

## 📋 策略概述

本服务实时监控贵金属价格波动，当价格相对 5 分钟内的极值（最高/最低）波动超过设定阈值时，自动发送飞书预警消息。

### 消息合并策略（v2.0）

- **黄金类合并**：国际现货黄金(XAU) + 国内黄金(CN_AU) → 一条消息
- **白银类合并**：国际现货白银(XAG) + 国内白银(CN_AG) → 一条消息
- **合并窗口**：3 秒内同时触发的预警会合并
- **涨跌不一致**：国内国际涨跌方向不同时，Header 使用白色

## 🎯 核心监控逻辑

### 1. 数据采集与合并策略

| 项目 | 配置 |
|------|------|
| 采集间隔 | **10 秒** |
| 数据保留时长 | **5 分钟** |
| 数据点数量 | **30 个** |
| **合并等待时间** | **3 秒** |

### 消息合并规则

```
触发预警
    ↓
等待 3 秒 ←── 同类别其他品种也触发？
    ↓              ↓
是 → 合并发送     否 → 单独发送
    ↓
判断涨跌方向
    ↓
同向（都涨/都跌）→ 红/绿 Header
反向（一涨一跌） → 白色 Header
```

**合并示例**：

| 场景 | 国内黄金 | 国际黄金 | 结果 |
|------|----------|----------|------|
| 同涨 | 涨 0.6% | 涨 0.8% | 🔸 黄金上涨预警 📈（红色） |
| 同跌 | 跌 0.7% | 跌 0.5% | 🔸 黄金下跌预警 📉（绿色） |
| 反向 | 涨 0.6% | 跌 0.3% | 🔸 黄金价格波动 ⚡（白色） |
| 反向 | 跌 0.5% | 涨 0.4% | 🔸 黄金价格波动 ⚡（白色） |

```
时间轴：00:00 ── 00:10 ── 00:20 ── ... ── 04:50 ── 05:00
数据点：P1    ──   P2   ──   P3   ── ... ──  P29  ──  P30

当前时间：05:10
有效数据：00:10 ~ 05:10 的 30 个点（00:00 已被移除）
```

### 2. 对比方式

**旧逻辑（已废弃）**：
- 当前价格 vs **5 分钟前的价格**
- 缺点：可能错过期间的大幅波动

**新逻辑（当前使用）**：
- 当前价格 vs **5 分钟内的极值**（最高价/最低价）
- 优点：捕获区间内的最大波动

```
5 分钟数据序列：[10.0, 10.2, 9.8, 10.5, 10.1, 9.9, 10.3]
                                  ↑               ↑
                                最高价          当前价
                                  
旧逻辑：10.3 vs 10.0 = 3% 波动
新逻辑：10.3 vs 9.8 = 5.1% 波动（捕获真实最大涨幅）
```

## 📊 预警触发条件

### 1. 上涨预警

**触发条件**：当前价格相对 **5 分钟内最低价** 上涨超过阈值

```javascript
涨幅 = (当前价格 - 5分钟内最低价) / 5分钟内最低价

if (涨幅 >= 阈值) {
  触发上涨预警（红色）
}
```

**示例**：
```
5 分钟内最低价：500.00 （发生在 2 分钟前）
当前价格：      505.00
涨幅：          (505 - 500) / 500 = 1.0%

黄金阈值：0.5%
1.0% > 0.5% → ✅ 触发上涨预警
```

### 2. 下跌预警

**触发条件**：当前价格相对 **5 分钟内最高价** 下跌超过阈值

```javascript
跌幅 = (5分钟内最高价 - 当前价格) / 5分钟内最高价

if (跌幅 >= 阈值) {
  触发下跌预警（绿色）
}
```

**示例**：
```
5 分钟内最高价：520.00 （发生在 3 分钟前）
当前价格：      517.00
跌幅：          (520 - 517) / 520 = 0.58%

黄金阈值：0.5%
0.58% > 0.5% → ✅ 触发下跌预警
```

## 🎛️ 阈值配置

### 按品种分类

| 类别 | 品种代码 | 品种名称 | 预警阈值 | 说明 |
|------|----------|----------|----------|------|
| **黄金类** | XAU | 国际现货黄金 | **0.5%** | 波动相对较小，阈值较高 |
| | CN_AU | 国内黄金 | **0.5%** | 同上 |
| **白银类** | XAG | 国际现货白银 | **0.03%** | 波动相对较大，阈值较低 |
| | CN_AG | 国内白银 | **0.03%** | 同上 |

### 阈值设计理由

| 品种 | 日内正常波动 | 阈值设定 | 触发频率 |
|------|-------------|----------|----------|
| 黄金 | ~1-2% | 0.5% | 中等（约每天 2-5 次） |
| 白银 | ~2-4% | 0.03% | 较高（约每天 5-10 次） |

## ⏱️ 冷却机制

### 冷却规则

- **冷却时长**：5 分钟
- **冷却对象**：同一品种（XAU/XAG/CN_AU/CN_AG）
- **冷却效果**：冷却期内即使触发条件也不发送消息

### 冷却示例

```
时间线：
09:00:00  黄金上涨 0.6% → 发送飞书 ✅
09:02:00  黄金上涨 0.7% → 冷却中，不发送 ❌
09:05:30  黄金上涨 0.8% → 冷却结束，发送飞书 ✅
09:08:00  黄金下跌 0.6% → 冷却中，不发送 ❌
```

## 🔄 完整监控流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      监控流程图（含合并）                         │
└─────────────────────────────────────────────────────────────────┘

[开始]
   │
   ▼
[每 10 秒采集价格]
   │
   ▼
[保存到 5 分钟环形缓冲区]
   │
   ▼
[计算 5 分钟内极值]
   ├─ 最高价
   └─ 最低价
   │
   ▼
[计算涨跌幅]
   ├─ 当前 vs 最低价 = 涨幅
   └─ 当前 vs 最高价 = 跌幅
   │
   ▼
[是否超过阈值?]
   │      ├─ 黄金 0.5%?
   │      └─ 白银 0.03%?
   │
   ├── 否 ──→ [继续监控]
   │
   ▼
[是否冷却中?] (5分钟)
   │
   ├── 是 ──→ [跳过，不发送]
   │
   ▼
[进入待发送队列]
   │
   ▼
[启动 3 秒合并窗口]
   │
   ├─ 3秒内同类别触发？
   │      ├─ 是 → [合并发送]
   │      │         ├─ 同向 → 红/绿 Header
   │      │         └─ 反向 → 白色 Header
   │      │
   │      └─ 否 → [单独发送]
   │
   ▼
[记录冷却时间]
   │
   ▼
[继续监控]
```

## 📊 策略优势

1. **及时性**：10 秒采集，秒级发现异常波动
2. **全面性**：对比极值而非单点，捕获真实最大波动
3. **差异化**：黄金/白银不同阈值，符合各自波动特性
4. **防骚扰**：5 分钟冷却 + 消息合并，避免消息轰炸
5. **可视化**：颜色区分涨跌（红/绿/白），一眼识别趋势
6. **信息聚合**：国内国际价格同屏展示，便于对比分析

### 消息合并的优势

| 对比项 | 合并前 | 合并后 |
|--------|--------|--------|
| 消息数量 | 4 条（各品种独立） | 2 条（黄金+白银） |
| 信息密度 | 分散，需逐个查看 | 集中，一目了然 |
| 关联分析 | 难对比国内外价差 | 同屏展示便于对比 |
| 骚扰程度 | 高 | 低 |
